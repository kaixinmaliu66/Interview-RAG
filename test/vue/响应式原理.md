## 一、Vue2 响应式原理特点

Vue2 的响应式核心是 **数据劫持 + 依赖收集 + 派发更新** ，整体流程：

1. **observe 观察**

   入口 `observe` 函数创建 `Observer` 实例，**递归遍历**对象 / 数组所有层级，把数据转为可观察对象。
2. **数据劫持**

   通过 `defineReactive` 使用 `Object.defineProperty`，对**每个已有属性**劫持 `get` / `set`。

   属于 **属性级劫持** 。
3. **依赖收集与更新**

   * `get` 中 `dep.depend()` 收集 `Watcher`
   * `set` 中 `dep.notify()` 通知更新
     依赖体系： **Dep + Watcher** 。

 **缺陷** ：

* 无法监听**对象新增 / 删除属性**
* 无法监听**数组下标赋值、length 修改**
* 初始化 **全量递归** ，性能开销大
* 只能劫持普通对象，对 Map/Set 等支持差

---

## 二、Vue3 响应式原理特点

Vue3 基于 **Proxy 代理**重构，同样遵循劫持、收集、更新，但机制完全不同：

1. **observe 观察**

   入口 `reactive`，内部 `createReactiveObject` 创建代理对象。

   采用 **惰性代理** ：只有访问到子对象时才递归响应式，不再一开始全量遍历。
2. **数据劫持**

   使用 `Proxy` 代理 **整个对象** ，拦截 `get/set/deleteProperty/has/ownKeys` 等 **全操作** 。

   属于 **对象级拦截** 。
3. **依赖收集与更新**

   * `get` 时 `track(target, key)` 收集 `effect`
   * `set/delete` 时 `trigger(target, key)` 触发副作用
     依赖体系： **effect + track + trigger** ，更通用、更解耦。

---

## 三、Vue3 相比 Vue2 的核心优化

1. **监听范围更完整**

   支持对象新增 / 删除属性、数组下标 /`length`、Map/Set/WeakMap 等。
2. **性能大幅提升**

   * 惰性递归，避免初始化深度遍历开销
   * Proxy 原生效率更高，无需重写数组方法
3. **架构更合理**

   * 响应式与渲染解耦，可独立使用 `@vue/reactivity`
   * 依赖模型更通用，支持自定义副作用
4. **行为更标准**

   配合 `Reflect` 保证 `this` 指向正确，边界情况处理更稳定。

---

# 最终收口一句话（专业简洁）

**Vue2 是递归 observe + defineProperty 属性劫持，Dep/Watcher 管理依赖，有监听盲区与性能问题；

Vue3 是 Proxy 对象代理 + 惰性 observe，track/trigger 管理 effect，实现了全覆盖、高性能、更健壮的响应式系统。**
