面试时聊 Vue-Router，不能只停留在“怎么配置路径”这种基础操作上。面试官更看重的是：**它是如何与 Vue 的响应式系统深度耦合，从而实现“无刷新渲染”的。**

我们可以从**宏观机制、核心模式、导航守卫**三个维度来彻底拆解。

---

## 1. 宏观机制：Vue-Router 的“三位一体”

Vue-Router 的本质是一个  **Vue 插件** 。它的核心逻辑可以概括为：**监听 URL 变化 **→** 匹配路由表 **→** 触发组件更新。**

* **响应式变量 `current`** ：Router 内部通过 `reactive`（Vue 3）或 `util.defineReactive`（Vue 2）维护了一个指向当前路径的对象。
* **`<router-view>` 组件** ：这是一个功能性组件，它的渲染函数（render）强依赖于上面的 `current` 变量。
* **闭环流程** ：当 URL 改变时，Router 会修改 `current` 的值。由于 Vue 的响应式机制，依赖这个变量的 `<router-view>` 会 **自动触发重新渲染** ，从而把旧组件替换成新组件。

---

## 2. 核心模式：它是如何拦截 URL 的？

面试必考点： **Hash 模式 vs. History 模式** 。

### ① Hash 模式 (默认)

* **原理** ：利用 URL 中 `#` 及其后面的部分。
* **机制** ：由于 `#` 后面内容的改变**不会**触发网页向服务器发送请求，它完全由前端控制。
* **监听** ：通过浏览器原生事件 `window.addEventListener('hashchange', ...)` 来捕获路径变化。
* **优点** ：兼容性极好（老版本 IE 都行），不需要服务器做任何配置。

### ② History 模式 (主流)

* **原理** ：利用 HTML5 提供的  **History API** 。
* **机制** ：
* 通过 `history.pushState` 和 `history.replaceState` 来修改浏览器地址栏，而 **不引起页面刷新** 。
* 通过 `popstate` 事件来监听用户点击“前进/后退”的操作。
* **致命伤与解决** ：因为 History 模式修改的 URL 看起来像真实路径（如 `/user/info`），如果用户刷新页面，浏览器会真向服务器请求这个路径。如果服务器没配置，就会报  **404** 。
* **后端配合** ：服务器（Nginx/Node）必须配置： **“无论请求什么路径，如果找不到文件，统一返回 `index.html`”** 。

## 3. 导航守卫：生命周期的延伸

守卫不仅仅是“权限校验”，它们是路由跳转过程中的 **生命周期钩子** 。

### 完整的导航解析流程（面试加分项）：

如果面试官让你说一遍跳转过程，按照这个顺序：

1. **导航被触发** 。
2. 在失活的组件里调用 `beforeRouteLeave`（我要走了，存个草稿？）。
3. 调用全局前置守卫 `beforeEach`（你要去哪？有权限吗？）。
4. 在重用的组件里调用 `beforeRouteUpdate`（参数变了，组件没变）。
5. 在路由配置里调用 `beforeEnter`（进这个门之前检查下）。
6. 解析异步路由组件。
7. 在被激活的组件里调用 `beforeRouteEnter`。
8. 调用全局后置钩子 `afterEach`。
9. **触发 DOM 更新** 。

---

## 4. Vue 3 中的重大进化

如果你在面 Vue 3 岗位，记得提这几点：

* **移除 `*` 通配符** ：现在必须使用正则（如 `/:catchAll(.*)*`）来匹配 404。
* **Composition API 支持** ：引入了 `useRouter` 和 `useRoute` 钩子，解决了在 `setup` 中拿不到 `this.$router` 的尴尬。
* **底层的解耦** ：Vue 3 的 Router 更加模块化，路由历史记录的管理（History Driver）和路径匹配逻辑完全分开，这让它更容易支持 **服务端渲染（SSR）** 。


## 总结话术（面试总结）

> “Vue-Router 的核心是 **将 URL 状态响应式化** 。
>
> 它通过 **Hash** 或 **History API** 劫持地址栏变化，而不触发浏览器刷新；随后通过内部维护的 **响应式路由对象** ，驱动 `<router-view>` 组件进行局部替换。
>
> 它的强大之处在于提供了一套完整的 **导航守卫体系** ，让我们能在组件渲染前，精准地控制数据预取、权限拦截等复杂的业务逻辑。”
>
