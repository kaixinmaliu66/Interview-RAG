## 核心总结：从“全量遍历”到“编译辅助的靶向更新”

### 1. 宏观策略的改变

* **Vue 2 (全量 Diff)** ：Vue 2 的渲染器是“盲目”的。无论模板中静态内容占比多大，当数据变化时，它都会从根节点开始，**递归遍历**整棵虚拟 DOM 树进行新旧对比。其性能开销与**整个模板的节点数**成正比。
* **Vue 3 (动静结合)** ：Vue 3 引入了编译时优化（Compiler-informed）。编译器在生成渲染函数时会分析模板，区分哪些是动态的、哪些是静态的。Diff 的性能开销只与**动态内容的数量**有关。

---

### 2. 算法细节的进化 (面试重点)

#### **Vue 2：双端 Diff 算法**

* **逻辑** ：通过“旧前、旧后、新前、新后”四个指针向中间靠拢。
* **优势** ：通过四种假设匹配（头头、尾尾、头尾、尾头），能高效处理简单的节点位移（如列表反转或首尾插入）。
* **局限** ：遇到复杂的乱序排列时，会回退到遍历映射表的暴力匹配，且无法识别并跳过静态节点。

#### **Vue 3：快速 Diff 算法 (Quick Diff)**

* **预处理** ：先进行从头到尾、从尾到头的“全量匹配”，排除掉两端完全相同的节点，缩小比对范围。
* **LIS（最长递增子序列）** ：针对中间的乱序部分，Vue 3 会根据新旧节点的索引关系生成一个数组，并利用 **LIS 算法** 找出其中不需要移动的“最长稳定序列”。
* **性能** ：只对不在序列中的节点进行 DOM 移动。这保证了在复杂场景下， **DOM 的操作次数是最少的** 。

---

### 3. 三大关键优化点 (杀手锏)

#### **① Patch Flags (动态标记)**

* Vue 3 在创建 VNode 时，会根据内容打上二进制标记（如：只有文本动是 1，只有类名动是 2）。
* **效果** ：Diff 时直接通过位运算（`&`）判断需要更新什么，实现 **靶向更新** ，跳过所有静态属性比对。

#### **② Hoisting (静态提升)**

* Vue 3 编译器会将模板中的静态节点、静态属性对象提升到渲染函数之外。
* **效果** ：这些 VNode 只在初始化时创建一次，更新时直接复用引用。不仅减少了内存占用，还让 Diff 算法直接跳过这些节点。

#### **③ Block Tree (块级树)**

* Vue 3 将模板按结构拆分为多个“块（Block）”。每个 Block 内部会维护一个扁平的数组，只记录其下属的动态节点。
* **效果** ：Diff 时不再递归整棵树，而是直接遍历这个扁平数组。这让虚拟 DOM 的复杂度从 **$O(Total Nodes)$** 降到了 **$O(Dynamic Nodes)$**。

---

### 4. 最终陈述（面试官最想听的总结）

> “总的来说，Vue 2 的虚拟 DOM 是纯运行时的算法，而 Vue 3 是 **‘编译+运行时’** 的完美结合。
>
> Vue 3 通过编译器为运行时留下了‘线索’（Patch Flags、Hoisting），配合更先进的 LIS 算法，使得它在处理大规模动态列表和复杂模板时，性能比 Vue 2 有了质的飞跃。这种从‘关注整体’到‘精准定位’的演进，是 Vue 3 性能领先的核心原因。”
>
