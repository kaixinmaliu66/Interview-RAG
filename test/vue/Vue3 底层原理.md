## Vue3 底层原理全链路口述版

### 一、 编译器阶段 (Compiler)：从字符串到可执行函数

“Vue 的编译阶段是一个将 HTML 模版‘工程化’的过程，核心目标是生成高性能的 `render` 函数。它分为三个环环相扣的步骤：”

#### **1. Parse（深度解析阶段）**

“这是最硬核的部分。Vue 并不是简单的正则替换，而是实现了一个 **有限自动机（FSA）** 。

* **扫描机制：** 编译器维护一个 `index` 指针，通过 `while` 循环不断消费 `template` 字符串。它根据当前字符特征切换状态，比如遇到 `<` 且匹配字母，状态机从 `Data` 切换到 `TagOpen`。
* **双向构建：** 配合一个**解析栈（Ancestors Stack）**来处理嵌套。解析到 `<div>`（开始标签）时入栈，解析到 `</div>`（结束标签）时出栈。栈顶元素始终是当前节点的父级，这样就能在消费字符串的同时，递归地构建出  **AST（抽象语法树）** 。
* **细节处理：** 此时会处理 HTML 实体转义、空白符压缩，并记录每个节点的 `loc`（行号列号），这也是 Vue 报错能精准定位的原因。”

#### **2. Transform（转化与优化阶段）**

“这是 Vue3 性能飞跃的关键。它会对 AST 进行二次加工：

* **指令转换：** 将 `v-if` 转化为逻辑三元表达式，将 `v-for` 转化为 `renderList` 循环。
* **静态提升（Hoisting）：** 编译器会识别出那些永远不会变的静态节点，把它们提到 `render` 函数外部。这样每次重新渲染时，这些 VNode 都会被复用，而不是重新创建。
* **性能补丁标记（Patch Flag）：** 这是最关键的。比如一个 `<div>` 只有文本是动态的，Transform 就会给它打上 `TEXT (1)` 的位运算标记。
  * *例子：* `<div :id="id">{{ msg }}</div>` 会被标记为 `TEXT | PROPS (9)`，告诉 Diff 算法：‘你待会儿只看 ID 和内容就行，其他的不用管’。”

#### **3. Generate（代码生成阶段）**

“最后，将优化后的 AST 转化为代码字符串。它会注入运行时需要的助手函数（如 `_createVNode`, `_toDisplayString`），通过 `new Function` 包装，生成最终的 `render` 函数。”

### 二、 响应式阶段 (Reactivity)：基于 Proxy 的精准代理

“当 `render` 函数执行时，会触发对数据的访问，这就进入了响应式系统：”

* **懒代理（Lazy Proxy）：** “Vue3 使用 `Proxy` 拦截。与 Vue2 一上来就递归 `defineProperty` 不同，Vue3 是 **按需代理** 。起初只对根对象做 Proxy，只有当你真正访问深层属性 `data.a.b` 时，它才会在 `get` 拦截里对下一层进行代理。这极大地减少了初始化时的 CPU 和内存开销。”
* **依赖收集（Track）：** “当 `render` 函数读取数据时，Proxy 的 `get` 会触发 `track`。它内部维护了一个庞大的 `WeakMap`：`target -> key -> [effects]`。它把当前的渲染函数（effect）记录在对应的属性下。”
* **触发更新（Trigger）：** “当你修改数据触发 `set` 时，会执行 `trigger`，从 `WeakMap` 里找到该属性对应的所有 `effect` 批量执行，通知组件重新渲染。”

---

### 三、 运行时阶段 (Runtime)：靶向更新的 Diff

“最后是闭环：数据变了，`render` 函数带着 Patch Flag 重新运行，生成新的虚拟 DOM。”

* **靶向更新：** “在 Diff 阶段，Vue3 不再像 Vue2 那样进行全量树遍历。它会直接跳到带有 **Patch Flag** 的节点。如果是 `TEXT` 标记，就只比对 `textContent`；如果是 `CLASS` 标记，就只更新类名。这种‘定向打击’配合静态提升，让 Vue3 的更新性能几乎不受组件模板大小的影响，只受动态节点数量的影响。”
