## 1. 浏览器的渲染流水线（关键上下文）

浏览器渲染页面大致遵循这个过程：

1. **解析 HTML** 生成 DOM 树。
2. **解析 CSS** 生成 CSSOM 树。
3. **合并**两者生成  **Render Tree（渲染树）** 。
4. **Layout（布局/回流）** ：计算每个节点在屏幕上的确切位置和大小。
5. **Paint（绘制/重绘）** ：将节点的几何属性转换为屏幕上的像素（背景、颜色、边框等）。
6. **Composite（层合成）** ：将不同的层合并并显示在屏幕上。

---

## 2. 什么是回流（Reflow）？

**回流**是“计算布局”的过程。当 DOM 元素的 **几何属性** （尺寸、位置）发生改变时，浏览器需要重新计算该元素及其受影响元素在页面中的位置。

* **特点：** 开销极大。因为一个元素的改变可能会触发其子元素、兄弟元素甚至整个页面的连锁反应。
* **触发场景：**
  * 页面首次渲染。
  * 浏览器窗口大小改变（Resize）。
  * 元素尺寸改变（`width`, `height`, `padding`, `margin`, `border`）。
  * 元素位置改变（`top`, `left`, `float`, `absolute` 等）。
  * DOM 结构变化（添加或删除可见的 DOM 元素）。
  * 获取某些特定属性（这会强制刷新队列）：`offsetWidth`, `clientHeight`, `scrollTop`, `getComputedStyle()`。

---

## 3. 什么是重绘（Repaint）？

**重绘**是“像素填充”的过程。当元素的**外观属性**发生改变，但不影响它在页面中的布局（不影响几何尺寸）时，浏览器会重新绘制该元素。

* **特点：** 开销比回流小，但频繁触发依然会影响性能。 **回流必定触发重绘** ，但重绘不一定触发回流。
* **触发场景：**
  * 背景颜色改变（`background-color`）。
  * 文字颜色改变（`color`）。
  * 边框样式（`border-style`）。
  * 可见性改变（`visibility: hidden` 触发重绘，而 `display: none` 触发回流）。

---

## 4. 两者的对比总结

| **特性**     | **回流 (Reflow / Layout)**    | **重绘 (Repaint)**          |
| ------------------ | ----------------------------------- | --------------------------------- |
| **触发原因** | 几何属性变化（位置、大小）          | 外观属性变化（颜色、背景）        |
| **性能影响** | **大** （涉及整个渲染树计算） | **较小** （仅局部像素更新） |
| **相互关系** | **回流必将引起重绘**          | **重绘不一定引起回流**      |

---

## 5. 如何优化？（面试加分项）

在开发中，我们的目标是 **尽量减少回流和重绘的次数** 。

### 1. 合并样式修改

避免一行行地修改样式，改用类名切换。

**JavaScript**

```
// ❌ 差：触发多次回流/重绘
el.style.width = '100px';
el.style.height = '100px';
el.style.margin = '10px';

// ✅ 好：一次性修改
el.className = 'new-style';
// 或者使用 cssText
el.style.cssText = "width: 100px; height: 100px; margin: 10px;";
```

### 2. 离线处理 DOM

如果需要进行大量的 DOM 操作，先让元素“脱离文档流”。

* 使用 `documentFragment` 批量插入。
* 操作前设置 `display: none`（一次回流），操作完再显示（一次回流）。

### 3. 避免频繁读取布局属性

浏览器通常会有个“渲染队列”优化策略，但如果你访问 `offsetHeight` 等属性，浏览器为了给你最新最准的值，会强制清空队列并立即触发回流。

**JavaScript**

```
// ❌ 差：循环中读取并修改
for(let i = 0; i < 10; i++) {
  el.style.left = el.offsetLeft + 5 + 'px';
}

// ✅ 好：缓存属性值
let curLeft = el.offsetLeft;
for(let i = 0; i < 10; i++) {
  curLeft += 5;
}
el.style.left = curLeft + 'px';
```

### 4. 利用 GPU 加速（合成层）

使用 `transform`（位移、缩放）和 `opacity` 不会触发回流和重绘，它们会触发 **合成（Composite）** ，由 GPU 完成，效率极高。这也是为什么动画建议优先使用 `transform: translate()` 而不是 `top/left`。
