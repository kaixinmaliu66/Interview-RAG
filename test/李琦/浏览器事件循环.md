# 浏览器事件循环（Event Loop）

## 面试口述完整版（带细节、例子、执行顺序、为什么）

**可直接背，深度足够，面试官问啥都能接住**

---

# 一、一句话核心

JS 是**单线程**语言，为了不阻塞主线程，把任务分成**同步任务**和 **异步任务** ，通过**事件循环 + 任务队列**调度执行。

---

# 二、任务分类（必考）

1. **同步任务** ：立即执行，进入**调用栈（Call Stack）**
2. **异步任务** ：不阻塞，进入 **任务队列** ，分两类：

* **微任务（Microtask）**
* **宏任务（Macrotask）**

## 优先级（死记）

**同步 > 微任务 > 宏任务**

---

# 三、微任务 vs 宏任务（必考清单）

### 微任务（优先级高）

* `Promise.then/catch/finally`
* `queueMicrotask()`
* `MutationObserver`
* `process.nextTick`（Node 独有，浏览器没有）

### 宏任务

* `script` 整体代码
* `setTimeout/setInterval`
* `I/O`（ajax、fetch）
* `UI rendering`
* `requestAnimationFrame`

---

# 四、事件循环执行规则（最标准流程）

1. 执行**调用栈**所有 **同步任务** ，直到清空。
2. 清空 **所有微任务队列** （一次性清空）。
3. 执行 **一个宏任务** （只取一个）。
4. 再次清空微任务。
5. 循环往复。

> 关键点：**微任务是每轮清空，宏任务一次只执行一个。**

---

# 五、经典面试题（必讲清楚顺序）

js

```
console.log('1') // 同步

setTimeout(() => {
  console.log('2') // 宏任务
}, 0)

Promise.resolve().then(() => {
  console.log('3') // 微任务
})

console.log('4') // 同步
```

**执行顺序：1 → 4 → 3 → 2**

### 为什么？

1. 先执行同步：1、4
2. 清空微任务：3
3. 执行一个宏任务：2

---

# 六、为什么微任务优先级更高？（面试官爱问）

* 微任务设计目的：**高优先级、轻量、尽快执行**
* 比如 Promise 要保证**异步但尽可能快**
* 微任务在 **渲染前执行** ，宏任务在**渲染后执行**
* 若微任务不清空，不会进入下一轮宏任务，也不会渲染

---

# 七、UI 渲染时机（重点）

**每一轮事件循环结束后，可能触发一次渲染：**

同步 → 微任务 → **渲染** → 宏任务

所以：

* 修改 DOM → 微任务里能拿到最新样式
* 宏任务里会 **先渲染再执行** ，会看到闪烁 / 延迟

---

# 八、为什么不能用 while 阻塞？

因为 JS 单线程，**调用栈不空 → 永远不执行异步任务**

页面卡死、点击无效、定时器不执行。

---

# 九、面试终极总结（60 秒口述版）

JS 是单线程，依靠事件循环实现异步。

任务分为 **同步、微任务、宏任务** ，优先级： **同步 > 微任务 > 宏任务** 。

执行流程是：先清空同步调用栈，再一次性清空所有微任务，然后执行一个宏任务，再清空微任务，循环往复。

微任务优先级高，在渲染前执行；宏任务一次一个，在渲染后执行。

这样既保证不阻塞主线程，又保证高优先级任务尽快执行。
