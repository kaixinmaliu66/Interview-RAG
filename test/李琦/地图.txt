# 🔥 一段式干货总结（面试直接口述，最显水平）
WebGIS 2D 以 Leaflet、OpenLayers 为主，底层基于 Canvas 2D，**无论瓦片图片还是 GeoJSON 矢量，所有计算、坐标映射、绘制全由 CPU 单线程处理**，
这就导致大数据量时极易卡顿，是核心性能瓶颈。优化思路围绕减轻 CPU 压力展开：通过点聚合减少绘制数量，用几何抽稀降低顶点计算量，
做视口动态加载只渲染当前视野数据，再通过 WebGL 插件把 2D 绘制交给 GPU 加速，超大规模数据则采用 MVT 矢量瓦片配合 GPU 渲染，
从根源解决卡顿。而 3D 地图如 Mapbox GL、Cesium 原生基于 WebGL，全流程 GPU 并行处理，流畅度和承载量远高于传统 2D，
但学习成本与资源占用更高，实际开发中会根据项目体量和需求选择 2D 轻量实现或 3D 高性能方案。
